# 日記ワークフロー

下書きテキスト: $ARGUMENTS

## 概要

スマホの音声入力で書いた荒削りな下書きを受け取り、以下を行う：

1. テキストを整形して日記ノートを作成（`■日記\YYYY\MM\YYYY-MM-DD.md`）
2. 必要に応じて購入物をアセットへ追記
3. 必要に応じてタスク・やることをタスクリストへ追記
4. 必要に応じて買い物を買い物リストへ追記
5. 必要に応じて将来調べるべき内容を気になることリストへ追記
6. 必要に応じて過去日記・周辺ノートへの関連リンクを付与
7. 必要に応じて関連ノート本文へ追記・更新（高確度のみ）
8. AIメモを日記末尾に追記（気になることの即答 + 日記へのひとことコメント）

## マルチエージェント方針

- **外側並列**: エントリごとに担当エージェントを起動（最大4並列）。`spawn_agent` → `send_input`（本文・日付・モード）→ `wait` → `close_agent`
- **内側並列**: 1エントリ内で整形・抽出・関連図担当を同時起動（最大6エージェント）。整形本文を基準に統合し重複除去
- **上限**: 同時12エージェント。同一ファイル書き込みは逐次化
- Phase間依存・同一ファイル競合を含む処理は並列化しない

## パス情報

- 日記格納先: `■日記\YYYY\MM\YYYY-MM-DD.md`
- frontmatter: `formatted: false` または **プロパティ未設定**（未整形） / `formatted: true`（整形済み）
- タスクリスト: `C:\Users\nedie\Obsidian\■メモ\📆タスクリスト.md`
- 買い物リスト: `C:\Users\nedie\Obsidian\■メモ\🛍️買い物リスト.md`
- 気になることリスト: `C:\Users\nedie\Obsidian\■メモ\🧭気になることリスト.md`
- アセットフォルダ: `C:\Users\nedie\Obsidian\■アセット\`
  - `1.ハードウェア.md`（IT機器・家電）/ `2.ソフトウェア.md`（買い切りソフト）/ `3.サブスクリプション.md`（テーブル形式）
  - `4.書籍リスト.md`（書籍）/ `5.スキル・資格.md` / `6.生活・住環境.md`（衣類・家具等）/ `9.その他.md`
- 見まわし対象フォルダ: `■メモ`・`■記事`・`■企画・プロジェクト`・`■リサーチ`・`■思考・アイデア`・`■リファレンス`・`■アセット`・`■家関係`・`■副業`

## 実行モードと動作ルール

- **標準**: 整形 + 高確度の他ノート追記 + 周辺ノート本文の追記/更新（常にこのモードで実行）
- 連携は高確度項目のみ。曖昧な候補はスキップ。同一内容の既存項目は追記しない
- 周辺ノート更新は追記優先・最小修正。全文リライトは行わない

## 引数の扱い

- 引数あり: `$ARGUMENTS` のテキストを下書きとして使用（今日の日付で `■日記\YYYY\MM\YYYY-MM-DD.md` に `formatted: false` で作成）
- 引数なし: 以下の手順で対象を収集する
  1. `■日記/*.md`（直下）— ObsidianのDaily Notesが作るファイル。`■日記\YYYY\MM\` に移動してから処理する
  2. `Grep` で `formatted: false` を `■日記/` 配下から検索
  3. `Glob` で `■日記/**/*.md` を取得し、各ファイルの frontmatter に `formatted` プロパティ自体が存在しないものも未整形として対象に含める（ObsidianのDaily Noteプラグインが自動作成したファイルには `formatted` が入らないため）
  - いずれもなければ空の日記ノートを今日の日付で新規作成する

## Phase 0: 入力解析

**複数日の判定**: 日付が2つ以上・区切り線（`---`/`===`）あり・複数日の出来事が並列 → いずれかで複数日モード。

**日付決定**: 明示日付を優先 → 「今日/昨日/一昨日」は相対計算 → 月不明な「X日」は直近日付 → 不明は今日の日付を使用。

複数日モード時は日付マーカー → 区切り線 → 自動推定の順で分割し、各エントリを担当エージェントで並列実行する。

## Phase 1: 下書きテキストの分析

### 1-1. テキスト整形

句読点補完・同音異義語修正・重複整理・文の区切り明確化。話し言葉の雰囲気は残す。

**`[[リンク]]` 化のルール（厳守）:** キーワードをリンク化する前に `Glob` でファイルの実在を確認する。ファイルが存在しないキーワードは絶対にリンク化しない。プレーンテキストのまま残す。

**情報の保全（厳守）:** 固有名詞・数量・金額・期日・意思計画は1つも落とさない。迷ったら残す。

### 1-2. 情報の抽出

確信度が低いものは「不確実」フラグを立て、追記しない。

- **購入物**: 「買った」「届いた」「入手した」等 → パス情報のアセットカテゴリに分類
- **タスク**: 「〜しなければ」「〜を予約」「〜を調べる」等 → 期日近い→平日/週末タスク、長期→対応セクション
- **買い物**: 「〜が切れた」「〜を補充」等 → 食料→スーパー / 日用品→日用品 / その他→一般
- **気になること**: 「あとで調べる」「比較したい」等 → テーマ名に正規化（優先度・根拠付き）、1日3件まで
  - **即答判定**: その場で1〜2文で答えられる軽い疑問は「即答可」フラグを立てる。深掘りが必要なものは従来通り気になることリストへ（→ `/research` で後日調査）
- **キーワード**: 過去日記・周辺ノート照合用（人名・場所・活動・関心事など）
- **周辺ノート更新候補**: 新事実・意思決定・次アクションを抽出。確信度「高」のみ自動更新対象

### 1-3. 内側並列（エントリ内分業）

整形・抽出・関連図担当を同時起動 → 整形本文を基準に統合し重複除去 → 書き込み担当へ渡す。

---

## Phase 2: 関連ノートの検索（標準モードのみ）

**日付上限ルール（厳守）**: このエントリの日付を `ENTRY_DATE` とする。Phase 2 全体を通じて、`ENTRY_DATE` より後の日付を持つ日記ファイル（ファイル名 `YYYY-MM-DD.md` が `ENTRY_DATE` を超えるもの）は一切参照・リンク・言及しない。未来の日記を読んでも、その内容はこのエントリのコンテキストに含めない。

1. **過去日記**: `Glob` で `■日記/**/*.md` → ファイル名の日付が `ENTRY_DATE` 以前のものだけを対象に → Phase 1 キーワードで `Grep` → 関連が高いものだけリンク
2. **周辺ノート**: 見まわし対象フォルダを `Glob` 探索 → キーワードで `Grep` → 本文一致・ファイル名・更新日でスコア付け → 高関連のみ `## 関連 > 周辺ノート` に追記。関連度上位で更新候補があるものを本文更新候補に選定
3. **関連図**: 検索結果を受けて関連図担当が mermaid でエンティティ図・ノート関係図を作成

---

## Phase 3: 実行モード確定

常に**標準モード**で実行する。確認は行わない。

## Phase 4: ファイルの作成・更新

- 標準: 4-1〜4-8 + 4-5-A を実行（4-5-A・4-7・4-8は必要時のみ）
- 文面のみ: 4-1のみ実行
- キャンセル: 何もしない
- 複数日: 4-1・4-7・4-8は各エントリごと、4-2〜4-6は全エントリ分まとめて一括処理

### 4-1. 日記ノートの移動と書き込み

`■日記\YYYY-MM-DD.md`（直下）にファイルがあれば `■日記\YYYY\MM\` に `mkdir -p` でフォルダ作成して移動。移動先に既存ファイルがあれば内容をマージ。

frontmatter がないファイルには `formatted: false` を付与。整形完了後に `formatted: true` に変更する。

`Edit` で整形済み本文を書き込む（元の下書き本文を整形後の本文で置換）。標準モードで追記先があれば `## 関連`（過去の日記 / 周辺ノート / 追記したノート / 関連図）セクションを追加（既存セクションがあればそこに追記）。

### 4-2〜4-5. 連携ファイル更新

- アセット: カテゴリ別ファイルの対応見出し末尾にリスト追記（サブスクはテーブルに行追加）
- タスクリスト: `📆タスクリスト.md` の対応セクション末尾に `- [ ] タスク` 追記
- 買い物リスト: `🛍️買い物リスト.md` の対応カテゴリに `- [ ] アイテム` 追記
- 気になること: `🧭気になることリスト.md` に `テーマ名 — 根拠（優先度 / 出典: [[日記パス]]）` を追記。同名既存項目には `/ 追記元:` を付記

### 4-5-R. 気になること — 追記後処理

気になること項目を1件以上追記した場合、**追記のみ**で完了する（確認・追加アクションは行わない）。

### 4-5-A. AIメモの作成

日記の `## AIメモ` セクションを末尾（`## 関連` の直前、なければ最末尾）に追記する。

**構成（すべて任意、該当するものだけ書く）:**

1. **気になること即答**: 「即答可」フラグが付いた項目について、AIの知識で1〜2文の簡潔な回答を書く。`WebSearch` は使わない（それは `/research` の役割）。確信が持てない回答は書かない
2. **ひとことコメント**: 日記の内容に対する率直なひとこと。ツッコミ・気づき・アドバイス・ダメ出し・過去日記との繋がりへの言及など。距離感の近い友人がチャットで返すくらいのトーン。持ち上げない。思ったことをそのまま書く。1〜3文程度

**フォーマット:**

```markdown
## AIメモ

- **〇〇って何だっけ？** → △△のこと。□□で使われてるやつ
- キーボードまた買ったのか。先月も悩んでたけど、そろそろ沼にハマってない？打鍵感は書いといたほうがいいよ、どうせ忘れるから
```

**ルール:**
- 日記の内容が薄い日（一行だけ等）は無理にコメントしない。セクション自体を省略してよい
- アドバイスやダメ出しはOK。ただし上から目線の説教や自己啓発ポエムはNG
- 過去日記の検索結果（Phase 2）を活用して「前にも〇〇って書いてたね」的な言及は積極的にやる
- **未来の日記を参照しない（厳守）**: `ENTRY_DATE` より後に書かれた日記の内容・出来事・結果は一切コメントに含めない。「この後〇〇になった」「翌日〇〇だったね」のような未来情報への言及は禁止。コメントは `ENTRY_DATE` 時点で知り得る情報のみで構成する

### 4-6〜4-8. 周辺ノート・逆参照・関連図

- 4-6（標準のみ）: 確信度「高」の候補を `Edit` で対象ノートに追記（既存見出し配下 or 末尾に `## 日記からの追記`。1ノート1〜3項目）
- 4-7（任意）: 関連性が十分高い過去日記に `## 関連 > 後日の日記` を逆参照追記
- 4-8（任意）: 関連図を日記の `## 関連 > 関連図` に mermaid ブロックで追記

## 注意事項

- 整形は意味を変えない。アセット追記は既存スタイルに合わせる。日記ファイルは上書きせず末尾追記
- 周辺ノート更新は追記中心。大幅改変は行わない。関連図は事実ベース（推測エッジなし）
- サブエージェント結果は親で統合し重複・矛盾を解消してから書き込む
