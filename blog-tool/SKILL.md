# ブログ管理統合ツール

操作/トピック: $ARGUMENTS

引数によってモードを自動判別する：

- **画像管理モード**: 引数が `images` または `dl`、あるいはURLを含む場合（例: `images 2026-02-15_記事`） → [画像管理モード](#mode-images) へ
- **記事作成モード**: それ以外の場合（トピック名など、例: `Rustdeskの使い方`） → [記事作成モード](#mode-post) へ

## 共通情報

### パス情報

- 記事MD: `C:\Users\nedie\Documents\Obsidian\■記事\article\`
- 画像: `C:\Users\nedie\Documents\Obsidian\■記事\Image\{記事フォルダ名}\`
- 画像保管リスト: `C:\Users\nedie\Documents\Obsidian\■記事\画像保管リスト.md`
- WordPress: https://serial.parallel.jp
- 記事フォルダ名 = MDファイル名から.mdを除いたもの（例: `2026-02-15_タイトル`）
- 画像ファイル名: `連番_説明.拡張子`（例: `01_エラー画面.png`）

### 画像リンク形式（標準Markdown）

```
![](../Image/{記事フォルダ名}/{ファイル名})
```

- **パスにスペースが含まれる場合は `%20` にエンコードする**（例: `練習1%20とにかく書いてみる`）
- **Obsidian独自形式 `![[...]]` は使わない**

### 画像保管リストへの自動追記ルール

記事に画像リンクを挿入したが、対応する画像ファイルがまだ存在しない場合：

1. 画像フォルダを作成する（`■記事\Image\{記事フォルダ名}\`）
2. `■記事/画像保管リスト.md` に以下の形式で**自動的に追記**する：

```markdown
---

## {記事タイトル}

保存先: `Image/{記事フォルダ名}/`

| #   | ファイル名    | 撮るもの                     |
| --- | ------------- | ---------------------------- |
| 1   | `01_説明.png` | 何を撮影するかの具体的な指示 |
| 2   | `02_説明.png` | 何を撮影するかの具体的な指示 |
```

3. ユーザーに「画像保管リストに{N}件追加しました」と通知する

## モード1: 記事作成 (Post)

### 文体ルール

このブログの文体を厳密に守ること：

- **常体（だ・である調）**が基本。「〜した」「〜できる」「〜すればOK」
- 段落は短く、1〜3文程度
- 見出しはシンプルで直接的（「方法」「原因」など短い語句）
- 試行錯誤の過程をそのまま素直に書く（「〜しようとしたが、エラー発生。」）
- 冗長な説明はしない。余分な装飾を排除した実用的な文体
- 「まとめ」セクションは不要
- カジュアルで親近感のある言葉遣い（「〜みたい」「〜けっこう」「〜したかった」）
- 絵文字は使わない

### 記事構成

1. 冒頭で何をしたいか/何が起きたかを短く書く
2. 問題や試行錯誤を見出しごとに区切って説明
3. 解決策や結果を書く
4. 必要に応じて補足情報

### ファイル形式

ファイル名: `YYYY-MM-DD_タイトル.md`（今日の日付を使用）

ファイルの先頭:

```
# タイトル

公開日: YYYY-MM-DD
カテゴリ: (Calligraphy / Diary / Furniture / Idea / Interior / Security / Software から選択)
URL: (空欄)

---

(本文)
```

### 手順

#### Phase 1: 記事作成

1. トピックについてWebSearchで最新情報を調べる（必要な場合）
2. 既存記事（`■記事\article\`）を数件読んで文体を再確認する
3. 文体ルールに従って記事を下書きする
4. 画像が必要な箇所に `![](../Image/{記事フォルダ名}/{連番}_{説明}.拡張子)` を仮挿入する（パス内スペースは `%20` エンコード）
5. 保存先に保存し、パスをユーザーに伝える

#### Phase 1.5: 画像保管リストに自動追記

記事内に画像リンクを挿入し、画像ファイルがない場合は[画像保管リストへの自動追記ルール](#画像保管リストへの自動追記ルール)に従って追記する。

#### Phase 2: WordPress投稿

6. AskUserQuestionでWordPressに投稿するか確認する
7. 投稿する場合、ブラウザで新規タブを作成し、WordPressログインページ（ユーザーに確認）へ移動
8. ログイン後、「投稿」→「投稿を追加」
9. タイトルを入力
10. **本文はコードエディタ（Ctrl+Shift+Alt+M）で入力する**
    - ブロックエディタのGUI操作は崩れやすいため避ける
    - 以下のテンプレートを使用する
11. カテゴリーを設定する（WordPressデータストアAPIを使用）
12. AskUserQuestionで公開 or 下書き保存かを確認し、保存する

#### Phase 3: 画像挿入（必要な場合）

画像がObsidianの画像フォルダにある場合：

1. ビジュアルエディタで画像を挿入したい位置にカーソルを置く
2. `/image` で画像ブロックを追加 → `find`でinputを特定 → `upload_image`ツールでアップロード
3. 代替テキスト・キャプション設定

## モード2: 画像管理 (Images)

WordPressの記事ページから画像をダウンロードし、Obsidian管理下に置く場合の手順。

### 手順1: WordPressから画像をDL

公開ページのHTMLから`<figure>`タグを抽出する（REST API無効のため）。

```javascript
const res = await fetch("https://serial.parallel.jp/?p={postId}");
const html = await res.text();
// ... (スクレイピングロジック)
```

- `wp-block-image`だけでなく全ての`<figure>`にマッチさせる
- 画像URLからリサイズサフィックス (`-XxY.ext`) を除去してフルサイズを取得
- 前後のテキスト（見出し、キャプション）を画像の説明として使う

### 手順2: HTML構造解析と配置特定

ブロック構造（h, p, img, list, code）の順序を取得し、MDの内容と照合して画像を適切な位置に挿入する。

### 手順3: MDファイルに画像リンク追加

1. 記事MDファイルを読む
2. 挿入位置を特定する
3. 対応する段落や見出しの後に `![](../Image/...)` を挿入する（パス内スペースは `%20` エンコード）
4. 編集内容を保存する

---

## 注意事項

- WordPressのREST APIは無効化されている
- 画像の説明が不正確な場合は、実際の公開ページの画像を目視確認するのが最善
- 画像管理ルールはGeminiワークフロー（`.agent/workflows/blog-images.md`）にも定義されている。変更時は両方を更新すること
